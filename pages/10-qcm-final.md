# 📝 ANSWERS

### Answers are on the following slides

**No peeking! 😉**

---

#### Answers 1–5

<div class="text-[9px]">

**Answer 1: B) `pwd`**
- `pwd`: Print Working Directory
- `ls`: lists files
- `cd`: changes directory

**Answer 2: C) `rwx`**
- 7 = 4 (read) + 2 (write) + 1 (execute)
- 6 = rw-, 5 = r-x, 4 = r--

**Answer 3: B) `root`**
- User `root` always has UID 0
- Superuser with full privileges

**Answer 4: B) `chown`**
- `chown`: change owner
- `chmod`: change permissions

**Answer 5: B) SIGTERM (15)**
- SIGTERM lets the process exit cleanly
- SIGKILL (9) terminates immediately

</div>

---

#### Answers 6–10

<div class="text-[9px]">

**Answer 6: B) `/etc/shadow`**
- `/etc/shadow`: hashed passwords (root-readable only)
- `/etc/passwd`: user account metadata

**Answer 7: B) `top`**
- `top`: live process display
- `ps`: snapshot at one instant

**Answer 8: B) 1**
- PID 1 is `init` or `systemd`
- Parent of all other processes

**Answer 9: A) `mount`**
- `mount`: attaches a filesystem
- `umount`: detaches it

**Answer 10: B) `/etc/fstab`**
- `/etc/fstab`: filesystem table
- Lists mounts at boot

</div>

---

#### Answers 11–15

<div class="text-[9px]">

**Answer 11: B) `du`**
- `du`: disk usage by file/directory
- `df`: free space by filesystem

**Answer 12: C) `127.0.0.1`**
- `127.0.0.1`: IPv4 loopback
- `::1`: IPv6 loopback

**Answer 13: B) 22**
- Port 22: SSH
- Port 21: FTP; port 23: Telnet

**Answer 14: B) `netstat` or `ss`**
- `ss`: Socket Statistics (preferred)
- `netstat`: legacy but common

**Answer 15: C) `apt`**
- `apt`: Advanced Package Tool (Debian/Ubuntu)
- `dnf`: RHEL/Fedora

</div>

---

#### Answers 16–20

<div class="text-[9px]">

**Answer 16: B) `apt update`**
- `apt update`: refreshes the **package index**
- `apt upgrade`: **installs** updates

**Answer 17: B) The first line**
- Shebang is `#!/bin/bash`
- Must be on line 1

**Answer 18: A) `for i in {1..5}; do echo $i; done`**
- Standard bash syntax
- `do` / `done` wrap the body

**Answer 19: B) `cron`**
- `cron`: recurring jobs
- `at`: one-shot scheduled job

**Answer 20: B) Daily at 2:00 a.m.**
- Cron fields: `minute hour day month weekday`
- `0 2 * * *` = minute 0, hour 2, every day

</div>

---

#### Answers 21–25

<div class="text-[9px]">

**Answer 21: B) `/var/log`**
- `/var/log`: standard log directory
- e.g. `syslog`, `auth.log`, `kern.log`

**Answer 22: B) `journalctl -f`**
- `journalctl -f`: follow (like `tail -f`)
- Live systemd journal

**Answer 23: B) Mandatory access control (MAC)**
- SELinux = Security-Enhanced Linux
- Modes: enforcing, permissive, disabled

**Answer 24: B) `setenforce 0`**
- `setenforce 0`: permissive
- `setenforce 1`: enforcing

**Answer 25: B) `fail2ban`**
- `fail2ban`: bans IPs after failed attempts
- Helps mitigate brute-force attacks

</div>

---

#### Answers 26–30

<div class="text-[9px]">

**Answer 26: C) SCP or SFTP**
- SCP: secure copy over SSH
- SFTP: SSH file transfer protocol

**Answer 27: B) `~/.ssh/`**
- `~/.ssh/`: per-user SSH directory
- `id_rsa`, `id_ed25519`: private keys

**Answer 28: C) `ssh-keygen`**
- e.g. `ssh-keygen -t ed25519` (recommended)
- Generates a key pair

**Answer 29: B) Nginx**
- Nginx: async, efficient
- Apache: older multi-process model

**Answer 30: B) `/etc/nginx/`**
- `/etc/nginx/nginx.conf`: main config
- `/etc/nginx/sites-available/`: site configs

</div>

---

#### Answers 31–35

<div class="text-[9px]">

**Answer 31: B) `nginx -t`**
- `nginx -t`: syntax test
- Always test before reload

**Answer 32: C) ACME**
- ACME: Automatic Certificate Management Environment
- Used by Certbot / Let's Encrypt

**Answer 33: B) `mysqldump`**
- `mysqldump`: SQL dump of a database
- e.g. `mysqldump -u root -p database > backup.sql`

**Answer 34: B) `start` runs now; `enable` at boot**
- `start`: runs the unit immediately
- `enable`: enables start on boot

**Answer 35: A) `/etc/systemd/system.conf`**
- Global systemd settings
- Units under `/etc/systemd/system/`

</div>

---

#### Answers 36–40

<div class="text-[9px]">

**Answer 36: B) `systemctl --failed`**
- Lists units in **failed** state
- Useful for troubleshooting

**Answer 37: B) Logical Volume Manager**
- LVM: flexible storage
- PV → VG → LV

**Answer 38: B) `ln -s`**
- `ln -s target link`: symbolic link
- `ln target link`: hard link

**Answer 39: B) `>` overwrites; `>>` appends**
- `>`: truncate and write
- `>>`: append

**Answer 40: C) `$HOME`**
- `$HOME`: home directory (`/home/user`)
- `$USER`: username

</div>

---

#### Answers 41–45

<div class="text-[9px]">

**Answer 41: B) `head file`**
- `head`: first 10 lines by default
- `tail`: last 10 lines

**Answer 42: D) Both B and C are correct**
- `find`: live filesystem search
- `locate`: database-backed search

**Answer 43: A) `gzip file`**
- `gzip`: compresses (creates `file.gz`)
- `gunzip`: decompress

**Answer 44: B) Makes the script executable**
- `+x`: add execute bit
- Needed for `./script.sh`

**Answer 45: B) `df`**
- `df`: disk free space by filesystem
- `df -h`: human-readable

</div>

---

#### Answers 46–50

<div class="text-[9px]">

**Answer 46: D) Both B and C are correct**
- `/tmp`: often cleared on reboot
- `/var/tmp`: persists across reboots

**Answer 47: B) `dmesg`**
- `dmesg`: kernel ring buffer
- Good for hardware/driver issues

**Answer 48: D) Both B and C are correct**
- `reboot`: immediate reboot
- `shutdown -r now`: same idea

**Answer 49: B) A background process**
- Daemon: long-running background service
- Examples: `sshd`, `nginx`, `cron`

**Answer 50: B) Use `sudo` for admin tasks**
- Least privilege
- `sudo` is auditable

</div>
